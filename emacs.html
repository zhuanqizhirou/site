TODO 14. Find/Replace Text in Directory

1.call dired to list files in dir, or call find-dired if you need all subdirectories.
2.type % m to mark the files you want.
3.type Q to cal dired-do-query-replace-regexp
4.type your find regex and replace string
5.for each occurence,type y to replace, n to skip, Ctrl-g to abort
6.type ! to replace all occurrences in current files without asking. N to skip all possible replacement for rest of te current file.
7.to do the replacement on all files without further asking,type Y.
8.call ibuffer to list all opened files.
9.Type * u to mark all unsaved files, type S to save all marked files. type D to close them all.

光标移动

上：C-p(previous).

下：C-n(next).

左：C-b(back).

右：C-f(forward).

移动到文件的开始处：M-<.

移动到文件的结尾处：M->.

移动到行首：C-a.

移动到行末：C-e(end).

向前移动一个词：M-f.

向后移动一个词：M-b.

向前移动一个句子：M-a.

向后移动一个句子：M-e.

向下滚动一屏：C-v(view).

向上滚动一屏：M-v.

将当前行置于屏幕中间：C-l.  2次 C-l 置于屏幕首，3次 C-l 置于屏幕末。

文件编辑

删除光标前字符：backspace（回车键上面那个）.

删除当前字符：C-d(delete).

删除光标后的一个词：M-d.

删除光标处到行末的字符：C-k(kill).

删除光标处到句末的字符：M-k.

使用melpa源:

把下面的代码放入配置文件中,启动emacs时自身检查列出的package是否已经安装,自动安装.
手动安装方法:
M-x list-packages,选择package,按i,标记安装,按x,执行安装.u,标记更新.

(require 'package)
(add-to-list 'package-archives
             '("melpa" . "http://melpa.org/packages/") t)
(when (< emacs-major-version 24)
  ;; For important compatibility libraries like cl-lib
  (add-to-list 'package-archives '("gnu" . "http://elpa.gnu.org/packages/")))
(package-initialize)

;; cl - Common Lisp Extension
(require 'cl)

;; Add Packages
(defvar my/packages '(
	       ;; --- Auto-completion ---
	       company
	       ;; --- Better Editor ---
	       hungry-delete
               web-mode
	       ;swiper
	       ;counsel
	       ;smartparens
	       ;; --- Major Mode ---
	       ;js2-mode
	       ;; --- Minor Mode ---
	       ;nodejs-repl
	       ;exec-path-from-shell
	       ;; --- Themes ---
	       ;monokai-theme
	       ;; solarized-theme
	       ) "Default packages")

(setq package-selected-packages my/packages)

(defun my/packages-installed-p ()
    (loop for pkg in my/packages
	  when (not (package-installed-p pkg)) do (return nil)
	  finally (return t)))

(unless (my/packages-installed-p)
    (message "%s" "Refreshing package database...")
    (package-refresh-contents)
    (dolist (pkg my/packages)
      (when (not (package-installed-p pkg))
	(package-install pkg))))

git教程再次学习

新学内容：
1. git add <file1> <file2> ;;一次添加多个文件到暂存区
2. git diff <file> ;;查看文件变更内容
？ 更准确地是git diff HEAD <file>,
?? 也不确定。优先git diff吧。
3. git log --pretty=oneline ;;单行显示版本变更记录
4. HEAD指向当前版本，HEAD^指向上一个版本，HEAD^^指向上一个版本的上一个版本，HEAD~2等价于HEAD^^
5. git reset --hard HEAD^ ;;退回上一个版本。 --hard表示保持与本地仓库一致，放弃本次修改。
或者使用 git reset <git id> ,至少4位。
6. cat readme.txt ;; 显示文件内容。
7. 返回到最新版本： 使用 git reflog 命令，查看commit记录，找到最新的版本id，git reset <id> 即可。
8. git reset HEAD ;;退回到本地仓库版本。？工作区中的修改？stage中的修改？
9. git reset <git id> , git reset --hard <git id> 的区别：
git reset <git id> :工作区中的修改保留，只是HEAD指向git id。
git reset --hard <git id> :工作区中的修改不保留，完全用git id版本覆盖。
？stage中的内容呢
stage中的内容，使用git reset HEAD后，是不存在的。当然，工作区中依然有。
当然，unstage的方式，本来就是 git reset HEAD <file>
use "git reset HEAD <file>..." to unstage;即取消放到暂存区操作。
use "git checkout -- <file>..." to discard changes in working directory;即取消工作区中的修改。 只是会撤销工作区的修改，已经stage的内容，不会被撤销。
rm <file> ;delete file
git rm <file> ;delete file,and stage.[same as: 1.rm file 2.git add file

第二阶段：分支等高级内容 

git branch dev ;创建dev分支
git checkout dev ;切换到dev分支
git checkout -b dev ;创建并切换到dev分支
在dev branch上进行的提交，切换回master后看不到。（如增加了文件）
git merge dev ;切换回master后，使用此命令合并dev分支。（使用了Fast-Forw
ard模式）
git branch ;查看分支
git branch -d dev ;删除dev分支
git log --graph --pretty=oneline --abbrev-commit ;使用图的形式，显示提交记录。尤其是有分支的情况。
git merge --no-ff -m 'commit log' dev ;合并分支，不使用fast-forward模式。当branch删除后，可以看出合并历史。
bug分支：
git stash ;保存当前工作目录。除了未被git管理起来的内容，无论是工作区还是暂存区（stage），内容都“消失”，status变得clean。
git stash list ;查看stash列表
git stash pop ;弹出stash继续工作。相当于git stash apply ,然后git stash drop
git stash apply stash@[0] ;恢复
git stash drop ;删除

无论是bug branch，还是feature branch，意义都在于，当前工作分支在dev上，而必须从master分支进行bug/feature。
所以虽然并没有提交，依然可以创建新branch，但是有问题的：
1. 当前dev分支，有未commit的内容。
2. git checkout -b feature
3. feature内容，和dev是一样的。
4. git checkout master ;必须commit，才可以切换分支。否则当前未commit的内容会丢失。
5. git add . ; git commit -m 'n' ;commit branch
6. git checkout dev
7. git branch -d feature ;欲删除该分支，因为应该从master分支上创建该分支。但出错：
error: The branch is not fully merged.
If you are sure you want to delete it, run 'git branch -D feature'
8. git branch -D feature ;feature branch被删除
9. git status ;发现原dev上的工作也丢失了！
